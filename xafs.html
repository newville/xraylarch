XAFS Analysis with Larch¶

One of the primary motivations for Larch was processing XAFS data. In fact, Larch is intended as version 2 of the Ifeffit engine, replacing and expanding all the XAFS analysis capabilities of that package. As of this writing, this replacement is well under way, but not complete.

As with most analysis of scientific data, XAFS Analysis can generally be broken into a few separate steps. For XAFS these are:

  1. Reading in raw data.
  2. Making corrections to the data, and converting to
  3. Pre-edge background removal and Normalization
  4. Interpreting normalized mu(E) as XANES spectra
  5. Post-edge background removal, conversion to
  6. XAFS Fourier Transform to
  7. Reading and processing FEFF Paths from external files.
  8. Fitting XAFS to FEFF paths.

As of this writing, many of these steps are supported in Larch. All the XAFS-specific functions are kept in the _xafs Group, and can be easily accessed as this is in the default search path. Note that many of the functions below take a group argument, which is the group to write resulting data into.If this is omitted, most of the functions below will return the most fundamental result, but this will be a minimal subset of the possible outputs.

_xafs.pre_edge(energy, mu, group=None, ...)¶

Pre-edge subtraction and normalization.

_xafs.find_e0(energy, mu, group=None, ...)¶

Guess E0 (, the energy threshold of the absorption edge) from the arrays energy and mu.

_xafs.autobk(energy, mu, group=None, rbkg=1.0, ...)¶

Determine the post-edge background function, , according the the “AUTOBK” algorithm, in which a spline function is matched to the low-R components of the resulting .

_xafs.ftwindow(k, xmin=0, xmax=None, dk=1, ...)¶

create a Fourier transform window function.

_xafs.xafsft(k, chi, group=None, ...)¶

perform an “XAFS Fourier transform” from to , using common XAFS conventions.
